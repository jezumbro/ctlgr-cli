ctlgr config

Manage config files and the registered catalog directory.

ctlgr config <subcommand>

Config file resolution

ctlgr walks up from the current working directory and uses the first config file it finds (first match wins):

  1. .ctlgr.local — personal overrides, not committed
  2. .ctlgr — project config, committed to version control
  3. ~/.ctlgr-cli/settings.json — global fallback
  4. ~/.ctlgr-cli/catalog/ — hardcoded default when no config is found

Subdirectories inherit the nearest ancestor config automatically.

Legacy migration

Older versions used .ctlgr.json / .ctlgr.local.json. On every invocation ctlgr automatically migrates these to .ctlgr / .ctlgr.local and removes the legacy file. Migration is skipped when the new-format target already exists.

Subcommands

init

Create a config file in the current directory registering a catalog directory.

ctlgr config init <path> [--local]
<path>
Catalog directory to register. Must exist and be a directory.
--local
Write .ctlgr.local instead of .ctlgr. Add .ctlgr.local to .gitignore.

Errors if the target file already exists. To change the path: delete .ctlgr and re-run. To revert to the default: delete .ctlgr.

# Committed project config
ctlgr config init ~/catalog

# Personal local-only override
ctlgr config init --local ~/my-catalog

list

Show the resolved catalog path. Prefixes (default) when falling back to the hardcoded default.

ctlgr config list

Config file schema

{
  "path": "/Users/you/catalog"
}
path
Absolute path to the catalog directory. All *.html and *.md files under it are searched recursively. Newly added files are picked up automatically.
