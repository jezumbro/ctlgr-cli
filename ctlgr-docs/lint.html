ctlgr lint

Validate catalog HTML files against style conventions. Default mode is non-destructive (--check); --write applies automatic fixes.

Usage

ctlgr lint [flags] [--file <file>...]

Flags

--file <file>, -f
Specific file(s) to lint (repeatable). Falls back to configured paths when omitted.
--write
Auto-fix violations where possible. Removes offending elements and attributes in-place. Exits 0 regardless of how many violations were fixed.

Exit codes

0
No violations found (check mode), or fixes applied (write mode).
1
One or more violations found in check mode.
non-zero
I/O or parse error.

Rules

no-style-blocks

Flag <style> elements. Inline style blocks couple presentation to catalog structure.

Violation

<article id="search">
  <style> h2 { color: red } </style>
  <h2>search</h2>
</article>

Fix (--write)

Remove the <style> element entirely.

no-inline-styles

Flag style="..." attributes on any element. Inline styles are presentation noise that does not belong in a catalog file.

Violation

<h2 style="font-weight: bold">search</h2>
<p style="color: grey">description</p>

Fix (--write)

Strip the style attribute from the element, leaving all other attributes intact.

Output format

Check mode prints one line per violation:

<file>:<line>: [<rule>] <snippet>

Write mode prints one line per fix applied:

<file>: fixed [<rule>] <snippet>

Examples

# Check a specific file
ctlgr lint --file catalog.html

# Check all registered catalog files
ctlgr lint

# Auto-fix a file
ctlgr lint --write --file catalog.html

# Auto-fix all registered files
ctlgr lint --write
